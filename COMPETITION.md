# 제3회 풍력발전량 예측 AI 경진대회 - BARAM 2026

> 대회 개요·평가 산식·규칙 정리 문서. 모델/에이전트가 과제 맥락을 참고하기 위한 레퍼런스.

## 배경

재생에너지 확대와 전력시장 변화에 따라, 풍력발전량을 안정적으로 예측하는 기술의 중요성이 커지고 있습니다.

- 풍력발전은 **풍속·풍향·기온** 등 기상 조건에 따라 발전량이 크게 달라지므로, 실제 운영 환경에서는 **기상예보 데이터**를 바탕으로 향후 발전량을 정밀하게 예측하는 역량이 필요합니다.
- 발전량 예측 정확도는 전력계통 운영 안정성뿐 아니라, **재생에너지 발전량 예측제도에 따른 정산금 확보**와도 직접 연결됩니다.

## 주제

**기상예보 데이터 기반 풍력발전량 예측 AI 모델 개발**

## 설명

기상청 기상예보 데이터와 풍력발전량 데이터를 활용하여, 특정 풍력단지(**3개 그룹**)의 향후 발전량을 예측하는 AI 모델을 개발합니다.

- 참가자는 제공된 학습 데이터로 예측 모델을 개발하고, 테스트 기간의 **시간 단위 풍력발전량 예측값**을 제출합니다.
- 예측 오차뿐 아니라 **재생에너지 발전량 예측제도에서의 정산금 획득 관점**도 함께 반영하여 모델의 실질적 활용 가능성을 평가합니다.

### 평가 단계
1. **1차 평가 (온라인 리더보드)** — 평균 예측오차율(1-NMAE)과 정산금획득률(FICR) 기반.
2. **2차 발표평가 (1차 상위 팀 대상)** — 과제 이해도, 기술 우수성, 문제 해결력, 적용 가능성 등을 종합 평가하여 최종 수상팀 선정.

---

## 1. 리더보드 평가 산식

```
총점(Score) = 0.5 × 평균 예측오차율(1-NMAE) + 0.5 × 정산금획득률(FICR)
```

> **평가 대상**: 실제 발전량이 **설비용량의 10% 이상**인 시간대만 대상으로 합니다.

### 1) 1-NMAE (평균 예측오차율)

```
그룹별 NMAE = 평균( |예측 발전량 - 실제 발전량| / 그룹 설비용량 )
1-NMAE      = 1 - (3개 그룹 NMAE 평균)
```

- 각 그룹별로 예측·실제 발전량의 차이를 그룹 설비용량으로 나누어 평균.
- **값이 높을수록 예측 오차가 작다**는 의미.

### 2) FICR (정산금획득률)

```
그룹별 FICR = 획득 정산금 / 이론상 최대 정산금
FICR        = 3개 그룹 FICR 평균
```

- 각 시간대별 NMAE 기준에 따라 획득 정산금을 계산하고, 이론상 최대 정산금과 비교.
- 시간별 정산금은 대회에서 제시한 기준표에 따라 산정.
- nMAE 구간 => 정산 기준  
  6% 이하 => 4원/kWh  
  6% 초과 ~ 8% 이하 => 3원/kWh  
  8% 초과 => 없음

## 2. Data Leakage 방지 규칙

본 대회는 **예측기준시점에 실제로 활용 가능했던 정보만** 사용하여 발전량을 예측하는 것을 원칙으로 합니다.

### 기본 원칙
- 각 예측값에는 해당 예측값의 **예측기준시점 이전에 생성·공개·확정되어 실제로 활용 가능했던 정보**만 사용 가능.
- 데이터의 사용 가능 여부는 **데이터가 가리키는 대상 시각이 아니라, 그 데이터가 생성·공개·확정되어 활용 가능해진 시각**을 기준으로 판단.
- 예측기준시점 이후에 관측·생성·공개·확정된 정보는, 예측 대상 시각에 대한 정보라도 사용 불가.

### 예보 데이터 사용 기준
- 기상예보 데이터는 **예보 대상 시각이 아니라 예보가 생성·공개되어 활용 가능해진 시각**을 기준으로 판단.
- 예측기준시점 이후에 생성·공개된 예보자료는 사용 불가.
- **예시**: 2025-01-13 10:00 발전량 예측 행의 예측기준시점이 2025-01-12 14:00인 경우, 2025-01-13 00:00에 생성·공개된 2025-01-13 10:00 대상 예보 데이터는 사용 불가.

### 실측 데이터 사용 기준
- 기상 관측자료 등 실측 데이터는 **예측기준시점 전에 이미 관측·공개·확정된 과거 자료**에 한해 사용 가능.
- 예측기준시점 이후에 관측·생성·공개·확정된 실측값, **사후 보정자료·재분석자료**는 사용 불가.

### 금지 정보
- 평가 구간의 실제 발전량, 실측 운영 데이터, 비공개 운영 데이터, 정답에 준하는 정보 사용 불가.
- 평가 데이터의 실제 정답(또는 준하는 정보)을 비공식적으로 수집·활용하는 모든 행위 금지 → **적발 시 즉시 실격**.
- 파생 변수·통계값·보간값·집계값 생성 시에도 **예측기준시점 이후의 정보가 포함되면 안 됨**.

### 소명 및 제재
- 참가자는 사용 데이터·전처리·학습·추론 과정이 규칙을 준수했음을 코드 검증 및 2차 평가에서 설명할 수 있어야 함.
- 규칙 위반이 확인되거나 소명이 충분치 않으면 평가 대상에서 제외될 수 있음.

## 3. 외부 데이터 사용 규칙

아래 기준을 **모두 충족**하는 경우에 한해 사용 가능.

### 사용 가능 기준
- 법적 제약이 없어야 함.
- 저작권·라이선스·개인정보·이용약관을 위반하지 않아야 함.
- **누구나 접근 가능한 공개 데이터**여야 함.
- Data Leakage 방지 규칙을 준수해야 함.

### 사용 제한 기준
- 특정 기관·기업·개인 등 제한된 주체만 접근 가능한 비공개/내부 운영 데이터는 사용 불가.
- 평가 데이터의 정답(또는 준하는 정보)을 포함/유추 가능한 데이터는 사용 불가.

### 제출 및 재현 의무
- 2차 평가 대상이 되면, 사용한 모든 외부 데이터 파일·출처·수집 방법·수집 시점·사용 기간·사용 변수·라이선스·전처리 코드를 성실히 제출.
- 외부 데이터는 **제출된 코드와 자료만으로 재현 가능**해야 함.
- 외부 데이터의 생성·공개·확정 시점이 각 예측값의 예측기준시점 이전임을 설명할 수 있어야 함.
- 출처·수집/전처리·활용 가능 시점·재현 가능성이 충분히 소명되지 않으면 평가 대상에서 제외될 수 있음.

- **2026년 7월 5일까지** 공식적으로 가중치가 공개된 오픈소스 모델만 사용 가능 (즉 2026-07-06 전).
- **오픈소스 라이선스 모델** = 가중치의 사용·수정·배포·재배포 및 **상업적 이용이 허용**되는 라이선스로 공개된 모델.
  - 단순히 가중치가 공개되어 있더라도 **비상업적/연구 목적/평가 목적 등으로 사용 범위가 제한된 모델은 사용 불가**.
- 사용한 모델·가중치·라이선스·다운로드 경로는 코드 검증 및 2차 평가에서 확인될 수 있음.
